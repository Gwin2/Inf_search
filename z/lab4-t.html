13. Связь с базами данных
на примере MySQL

Одной из самых приятных особенностей языка РНР является легкость, с которой программист на нем может общаться с базами данных. В этой главе в основном будет говориться о СУБД MySQL, но и с остальными базами данных РНР работает аналогично и с такой же легкостью. Из-за чего уделяется такое внимание именно MySQL? Потому что эта система управления базами данных соответствует духу РНР, она распространяется бесплатно и обладает достаточной мощностью для того, чтобы использовать ее для решения реальных задач. Кроме того, существуют версии MySQL для самых разных платформ.

Подключение к серверу базы данных

Перед тем как начинать работать со своей базой данных, вам нужно подключиться к серверу.

Для этого в языке РНР есть модуль mysqli. mysqli (или MySQL Improved)  – это расширение PHP, которое предоставляет возможность взаимодействия с базой данных MySQL. Оно предоставляет процедурный и объектно-ориентированный интерфейс для работы с базой данных. При процедурном стиле установка соединения с базой данных реализуется с помощью функции mysqli_connect(). Функция mysqli_connect() возвращает идентификатор подключения, если все прошло успешно. Этот идентификатор нужно сохранить в переменной и в дальнейшем пользоваться им для работы с сервером базы данных. В объектно-ориентированном стиле взаимодействие с базой осуществляется через работу с объектом класса mysqli.

Пример использования mysqli в объектно-ориентированном стиле:

$conn = new mysqli('hostname', 'username', 'password', 'database');

$conn->close();

При создании объекта $conn в конструктор класса передаются следующие необязательные параметры:

hostname – имя хоста на котором находится сервер базы данных;

username – имя пользователя под которым происходит подключение;

password – пароль пользователя для доступа к базе;

database – имя базы.

По умолчанию порт сервера MySQL – 3306, но если сервер был перенес на другой порт, его можно указать пятым параметром при вызове конструктора класса.

В случае, если какой-либо параметр не задан, то в конструктор подставляются значения по умолчанию, который описаны в конфигурационном файле ini.

Метод close() закрывает соединение с базой. Открытые непостоянные соединения MySQL и наборы результатов автоматически закрываются при уничтожении их объектов. Явное закрытие открытых соединений и освобождение наборов результатов не обязательно. Однако рекомендуется закрыть соединение, как только скрипт завершит выполнение всех своих операций с базой данных, если ему ещё предстоит большая обработка после получения результатов.

Выбор базы данных

Если при создании соединения с сервером параметр database был пропущен и возникла необходимость переключится с базы данных установленной по умолчанию на другую, то можно воспользоваться метод select_db(). Метод select_db() используется для выбора базы данных, с которой будет взаимодействовать текущее соединение. Этот метод позволяет указать конкретную базу данных, с которой будут выполняться SQL запросы, без необходимости включения имени базы данных в каждом SQL запросе.

$database = "sample";

$mysqli->select_db($database);

Обработка ошибок

Если при создании соединения с сервером MySQL возникают ошибки, их можно обработать с помощью метода connect_error.

if ($conn->connect_error) {

die("Соединение не установлено: " . $conn->connect_error);

}

Если соединение не установлено, скрипт завершится с выводом сообщения об ошибке подключения.

Если программа будет пытаться открыть несуществующую базу данных (db_2), то в таком случае функция die() выведет примерно такое сообщение:

Нельзя открыть db_2: Access denied for user: 'pGG@localhost' to database 'db_2'

Добавление данных в таблицу

Получив доступ к базе данных, мы можем добавлять информацию в ее таблицы. Рассмотрим это на примере. Представьте себе, что нам нужно создать узел, на котором пользователи сети могут покупать для себя доменные имена.

Мы создаем в базе данных sample таблицу с именем domains_brNN (где NN — номер бригады, например, 11, 12 и т.д.). Таблица содержит 4 поля: поле первичного ключа с именем id, значение которого будет автоматически увеличиваться при добавлении новых записей, поле domain, куда записана строка переменной длины типа VARCHAR, поле myname, содержащее фамилию пользователя, и поле mail, содержащее адрес пользователя.

Для создания этой таблицы была использована такая команда SQL:

create table domains_brNN
       (id INT NOT NULL AUTO_INCREMENT,
        PRIMARY KEY(id),
        domain VARCHAR(20),
        myname VARCHAR(10),
        mail VARCHAR(20)
       );

Для добавления данных в эту таблицу нам нужно сконструировать и выполнить запрос SQL. Для этого в классе mysqli есть метод query(). Этому методу нужно передать строку с запросом. Метод возвращает true, в случае успешного выполнения запроса, и false, если в запросе содержится ошибка или если вы не имеет права на выполнение такого запроса.

$sql = "create table domains_brNN
         (id INT NOT NULL AUTO_INCREMENT,
          PRIMARY KEY(id),
          domain VARCHAR(20),
          myname VARCHAR(10),
          mail VARCHAR(20)
         )";
// здесь NN – номер бригады

 

if (!$conn->query($sql)) {

    print("<p>Ошибка: ”.$conn->error);

}

Имейте в виду, что успешное выполнение запроса не обязательно подразумевает изменение данных в таблице. В программе из листинга 13.2 мы для простоты и краткости опустили один очень важный момент — нами не проверялись данные, введенные пользователем. А так поступать, вообще-то, не следует. Нельзя доверять пользователю в вопросах достоверности данных. Все данные, введенные им в форму, нужно тщательно проверять.

Нами проверяется существование переменных $domain, $myname и $mail. Если эти переменные определены, то мы можем быть уверены, что форма передана, и вызываем метод query() объекта $database.

Метод query() принимает один аргумент, который содержит SQL-запрос, построенный с использованием переменных $domain, $myname, $mail.

В конструкторе класса создается подключение к серверу MySQL, и если оно не происходит, то прекращаем выполнение программы, вернув сообщение об ошибке.

В методе query() у нас десть проверка возвращаемого значений после выполнения SQL-запроса. Если после выполнения SQL-запроса значение true, то мы можем быть уверены в том, что данные добавлены. Если нет, то это означает, что произошла ошибка. В таком случае выводим сообщение об ошибке на экран браузера. Описание этой ошибки содержится в свойстве $error класса mysqli, поэтому включаем ее в текст сообщения.

Если в операторе if мы обнаруживаем, что одна из переменных — $domain, $myname или $mail — не определена, то это означает, что данные не переданы пользователем. В таком случае вызываем метод writeForm(), который выводит форму на экран пользователя.

Относительно метода query() нужно помнить следующее: ее аргумент, т.е. строка запроса к базе данных, не должна содержать символа «;». Фактически это означает, что аргументом может быть только один SQL-запрос.


Следовательно, если требуется последовательно выполнить несколько SQL-запросов, необходимо столько же раз использовать mysqli_query(), как в следующем примере:

$query = "INSERT INTO domains_brNN (domain, myname, mail)
          values('aaa.com', 'aaa', 'aaa@mail.ru')";
$database->query($query);

$query = "INSERT INTO domains_brNN (domain, myname, mail)
          values('bbb.com', 'bbb', 'bbb@mail.ru')";
$database->query($query);

$query = "INSERT INTO domains_brNN (domain, myname, mail)
          values('ddd.com', 'ddd', 'ddd@mail.ru')";
$database->query($query);

Получение значения автоматически изменяемого поля

В предыдущих примерах мы добавляли записи в таблицу, не беспокоясь о значении поля id, которое автоматически увеличивалось при каждом создании новой записи. Если нам понадобится значение этого поля для некоторой записи, мы всегда можем его получить с помощью SQL-запроса. Однако как поступить, если это значение нам нужно немедленно? В классе mysqli есть свойство insert_id, которое хранит значение ключа последней добавленной записи.

Таким образом, если нам необходимо сообщить пользователю номер, присвоенный его заказу, мы можем для этого получить значение свойства insert_id непосредственно после добавления новых данных в таблицу. Таким образом добавим в наш класс новый метод

public function getOrderId() {

    return $this->connection->insert_id;

}

Поскольку в свойстве $this->connection у нас хранится объект класса mysqli, то мы обращаемся к его свойству insert_id.

Вызовем новый метод и отобразим информацию о номере заказа.

$query = "INSERT INTO $table(domain, myname, mail) VALUES('$domain', '$name', '$mail')";

    if ($database->query($query)) {

          $id = $database->getOrderId();

          print "Спасибо. Ваш номер - $id.

          Пожалуйста, используйте его в других запросах.";

     }

Доступ к информации

Мы уже умеем добавлять данные в базу данных, но необходимо еще и уметь их оттуда читать. Вы, наверное, догадываетесь, что для этого нужно сделать SQL-запрос типа SELECT, но как воспользоваться результатами этого запроса, т.е. возвращенными записями? При успешном выполнении запроса метод query()вернёт объект mysqli_result, который хранит информацию о результатах запроса.

Число записей, найденных в запросе

Узнать число записей, возвращенных в результате выполнения запроса SELECT, можно с помощью обращения к свойству num_rows объекта mysqli_result.

Результат запроса

После выполнения запроса SELECT и получения объекта mysqli_result вы можете в цикле просмотреть все записи, найденные в результате запроса. РНР создает для вас внутренний указатель, в котором записана позиция в наборе записей результата. Этот указатель автоматически перемещается на следующую позицию после обращения к текущей записи.

С помощью метода fetch_row()объекта mysqli_result можно для каждой записи получить массив, состоящий из ее полей. По достижении конца запроса метод fetch_row() вернет значение false.

Чтение отдельных полей

Когда вы, выполнив запрос SELECT, получили объект mysqli_result, то можете определить количество возвращенных полей с помощью обращения к свойству field_count объекта mysqli. Это свойство содержит целое число, равное количеству найденных полей.

public function getFieldCount() {

    return $this->connection->field_count;

}

Каждое поле в этом наборе имеет свой номер, начиная с нуля, и вы можете, указав номер поля, определить все его свойства, включая имя, тип, максимальную длину и флаги.

Для выяснения свойств поля передайте номер поля методу fetch_field_direct() объекта mysqli_result. Данный метод принимает число, обозначающее порядковый номер поля, а возвращает вам объект – переменную, хранящую в себе все свойства данного поля.

Чтобы получить имя поля, обратитесь к свойству name:

$result = $database->query($query);

$num_fields = $database->getFieldCount();

for ($x=0; $x<$num_fields; $x++) {

     $properties = $result->fetch_field_direct($x);

     print ($properties->name)."<br>\n";

}

Для того чтобы узнать максимальную длину поля, обратитесь к свойству length.

$result = $database->query($query);

$num_fields = $database->getFieldCount();

for ($x=0; $x<$num_fields; $x++) {

    $properties = $result->fetch_field_direct($x);

    print ($properties->length)."<br>\n";

}

Для того чтобы узнать флаги, обратитесь к свойству flags.

$result = $database->query($query);

$num_fields = $database->getFieldCount();

for ($x=0; $x<$num_fields; $x++) {

    $properties = $result->fetch_field_direct($x);

    print ($properties->flags)."<br>\n";

}

Точно так же можно выяснить тип поля.

$result = $database->query($query);

$num_fields = $database->getFieldCount();

for ($x=0; $x<$num_fields; $x++) {

    $properties = $result->fetch_field_direct($x);

    print ($properties->type)."<br>\n";

}

Обратите внимание, что для типа поля и флагов возвращается не их строковое название (пример – INT, CHAR, BOOL), а число, которое обозначает их название. Например, типу BOOL соответствует число 1.

В листинге 13.3 приведен пример выполнения запроса SELECT, который запрашивает все строки таблицы domains_brNN, затем определяет размер этой таблицы с помощью свойства num_rows и выводит на экран всю таблицу domains_brNN.

while ($a_row = $result->fetch_row()) {      // печатаем содержимое столбцов

     print "<tr>\n";

Метол fetch_row() возвращает массив.

После подключения к серверу базы данных мы с помощью метода query() посылаем запрос на сервер базы данных. Возвращенный результат запроса сохраняем в переменной $result. Потом нами будет использоваться эта переменная для обращения к результату запроса: в переменной $num_rows мы сохраняем количество записей в запросе, а в переменной $num_fields — количество полей (столбцов) в запросе. Затем, в цикле, в переменную $name заносим название $x–того поля таблицы и выводим это название в виде ячеек-заголовков <th>.

В условном выражении цикла while мы присваиваем переменной $a_row значение, возвращенное методом fetch_row(). Результатом операции присваивания является значение ее правого операнда, поэтому данное условное выражение имеет значение true, если метод fetch_row() вернет положительное число. Внутри цикла while мы просматриваем массив, записанный в переменной $а_row, и выводим каждый элемент этого массива на экран, обрамляя его тегами ячейки таблицы.

Кроме того, к полям записи можно обратиться по имени: метод fetch_assoc()возвращает ассоциативный массив, в котором в качестве ключа используются имена полей. В следующем примере используется эта функция.

print "<table style='border: 1px >\n";

while ($a_row = $result->fetch_assoc())

print "<tr>\n";

print "<td>$a_row[mail]</td>
       <td>$a_row[domain]</td>\n";

print "</tr>\n";

print "</table>\n";

 


Изменение данных

Данные в таблице можно изменять с помощью метода query() в сочетании с оператором UPDATE. Как и ранее, успешное выполнение оператора UPDATE не обязательно означает, что данные были фактически изменены. Для того чтобы узнать количество измененных данных в таблице, вам придется обратиться к свойству affected_rows объекта. Этим свойством можно пользоваться в сочетании с любым запросом, в результате которого данные могли быть изменены (UPDATE, INSERT, REPLACE или DELETE, но не SELECT!).

В листинге 13.4 приведен пример программы, позволяющей администратору базы данных изменять любые данные в поле domain таблицы domains_brNN.

Мы открываем соединение с сервером и выбираем базу данных как обычно. Потом проверяем существование переменных $domain и $id. Если они существуют, то мы строим SQL-запрос, в котором изменяем значение поля domain для тех записей, у которых значение поля id совпадает со значением переменной $id. Мы не получим сообщения об ошибке, если попытаемся воспользоваться несуществующим значением переменной $id или если значение поля domain и так совпадает со значением переменной. Просто в таком случае свойство affected_rows() вернет нам 0. Полученное значение (в данном примере это должна быть 1) мы выводим на экран браузера.

Нами выводится форма, с помощью которой администратор может вносить изменения. Обратите внимание на то, что мы опять используем метод query() для того, чтобы получить значение полей id и domain и использовать их в HTML-тексте для формирования списка. Администратор будет выбирать в этом списке данные, подлежащие изменению. Если администратор уже передал форму и выбранное значение id совпадает текущим полем, мы выводим в элемент OPTION строку SELECTED. Это делается для того, чтобы новое значение, выбранное администратором, было выделено в форме.

Важное замечание. Если аргумент метода query() является переменной, значение которой было передано из другой программы с помощью метода POST, и в этой переменной содержится SQL-запрос, в котором есть апострофы, например

$query = "SELECT * FROM domains where id='2' ";

то эти апострофы будут экранированы обратной косой чертой (\). Для того, чтобы можно было использовать значение переменной $query в методе query(), необходимо применить функцию stripslashes(), которая убирает все управляющие символы (в том числе и обратную косую черту перед апострофами) из указанной строки.

Получение информации о базе данных

До сих пор мы рассматривали функции, предназначенные для сохранения и чтения данных. Однако в РНР есть несколько функций для того, чтобы вы могли получить дополнительную информацию о базах данных, доступных в данном подключении.

Список баз данных

Список всех баз данных, доступных для данного подключения, можно получить с помощью специального запроса:

$db_list = $database->query("SHOW DATABASES");

После этого полученным списком таблиц можно пользоваться аналогично результату любого запроса. В частности, у $db_list можно вызывать метод fetch_row() и перебрать все имена баз данных.

Список таблиц в базе данных

Список всех таблиц, доступных для данной базы данных, можно получить с помощью специального запроса:

$table_list = $database->query("SHOW TABLES");

Если указанная база данных существует и вы имеет соответствующие права, то метод вернет объект mysqli_result, у которого можно вызвать метод fetch_row() или аналогичный ему.

В следующем примере показано, как можно получить список всех таблиц базы данных.

$table_list = $database->query("SHOW TABLES");

while ($tab_rows = $table_list->fetch_row())

       print "$tab_rows[0]<br>\n";

14. Включение файлов в документ

В каждый РНР-документ можно включить файл с помощью команды include(). После этого текст РНР-программы во включаемом файле будет исполняться точно так же, как если бы он был записан во включающем файле непосредственно. Этим удобно пользоваться для одной и той же программы во многих документах.

Предположим, вы написали очень ценную программу, которой можно и нужно пользоваться во многих документах. У вас есть возможность непосредственно вставить текст этой программы в каждый документ, где она необходима. Но в таком случае при обнаружении ошибки в программе или просто в случае ее модификации вам придется вносить изменения в каждом документе, использующем данную программу. Избежать этого позволяет оператор include(). Вы можете записать свою программу в один документ, а потом включать этот документ в другие. Функция include() требует одного аргумента — пути к файлу, который нужно включить в документ. В листинге 14.1 приведен пример того, как в простую РНР-программу включается содержимое файла, причем содержимое включаемого файла обрабатывается как РНР-программа.

При выполнении документа из листинга 14.1 строки, содержащиеся во включаемом документе, выводятся на экран. Причем, если вы хотите, чтобы содержимое этого файла обрабатывалось как РНР-программа, вы должны обрамлять его начальным и завершающим тегами РНР.

Включаемые файлы в РНР могут возвращать значения так же, как это делают функции. Использование оператора return прерывает выполнение включаемого файла, как это происходит с функцией. В листингах 14.3 и 14.4 приведен пример использования функции include(), возвращающей значение.

Команду include() можно использовать внутри условного оператора. В таком случае включаемый файл будет выполняться только при выполнении условия.

Например, в следующем фрагменте команда include() не будет выполнена.

$test = false;

if ($test)
   {
    include("a_file.txt");  // не будет включен
   }

Если команду include() использовать внутри цикла, то она будет выполняться при каждой итерации. В листинге 14.5 приведен пример того, как команда include() используется внутри цикла for. Обратите внимание на то, что при каждой итерации включается другой файл.
